# compress these types of messages
# We want to use run length encoding to compress the messages
# Output should be:
# T1: [ [ 3, 2 ], [ 5, 1 ], [3, 2], [2,0], [2,2] ]
# T2: [ 13, 0 ] ]
# T3: [[1,2], [3,0], [2,3], [3,0], [4,2] ]


# Program:
def run_length_encoding(message):
  result = []
  count = 1
  if len(message) > 0:
      
    for i in range(1,len(message)):
      if message[i] == message[i - 1]:
        count += 1
      else:
        result.append([count, message[i - 1]])
        count = 1
    result.append([count, message[-1]])
  return result

Run Length Encoding

Run Length Encoding (RLE) replaces each symbol in the message with a count and the symbol

So that may allows us to compress a message

Efficiency depends on the probability that we have long runs of symbols

Run Length Encoding

Good Example: [0,0,0,0,0,0,0,0] -> [[7,0]]

Bad Example: [0,1,2,3,4,5] -> [[1,0], [1,1], [1,2], [1,3], [1,4], [1,5]]

Lets each number is counted as a transmission unit (1U)

Good Example: 7U -> 2U: 2/7

Bad Example: 6U -> 12U: 12/6 = 2 (Worst Cade for RLE)

import random

def generate_message(p = 0, max_length = 10):
  msg = [0]
  for i in range(1, max_length):
    if ( random.random() < p ):
      msg.append( msg[-1] )
    else:
      msg.append( 1 - msg[-1] )
  return msg

print(f"Message:{generate_message(0,5)}")
print(f"Message:{generate_message(0.5,5)}")
